# Orbit Phase 2: 楽曲DB再設計（リリース中心）要件

- Issue: https://github.com/kikun-dev/personal-hub/issues/58
- 更新日: 2026-03-09

## 背景

既存の楽曲機能（`orbit_songs` 中心）は、今回の要件で必要な「リリース中心管理」「複数リリース再収録」「人物マスタ参照クレジット」「MV/衣装/特典映像」を十分に表現できない。
そのため、Phase 2 の楽曲領域はデータモデルとアプリ層を再設計する。

## 決定事項（仕様確定）

- 楽曲とリリースは M:N とし、中間テーブルでリリースごとの曲順を保持する。
- グループはリリースのみ保持する。楽曲のグループ要件は「紐づくリリース由来」で満たす。
- 人物マスタを新設し、作詞/作曲/編曲/振付/MV監督/衣装担当は人物参照で管理する。
- リリースタイプは `single` / `album` / `digital_single` / `other`。
- ナンバリングは `single` / `album` のみ必須（整数）。他タイプでは未設定必須。
- 楽曲時間は秒数（`int`）で保存する。
- MVは「存在する場合リンク必須」。監督/ロケ地/配信日/メモは任意。
- アートワーク・特典映像・MV・衣装は未設定可（空配列/NULL可）。
- 画像（リリースアートワーク、衣装画像）は Supabase Storage object path で保存する。
- 既存 `orbit_songs` 系は移行せず、置換（ゼロ再構築）する。

## 必須/任意

### リリース

- 必須: `title`, `group`, `type`
- 任意: `numbering`（ただし `single/album` では必須）, `releaseDate`, `artworkPath`, `bonusVideos`, `participants`

### 楽曲

- 必須: `title`, `release紐付け`
- 任意: `durationSeconds`, `credits`, `formation`, `mv`, `costumes`

## 機能要件

### 1. リリース管理

- リリースごとにタイトル、グループ、タイプ、ナンバリング、リリース日を管理できる。
- 版種ごとの特典映像を複数行で管理できる（例: Type-A, Type-B）。
- リリース参加メンバーを複数選択できる。
- リリース曲目アートワークを保持できる。

### 2. 楽曲管理

- 楽曲は1件以上のリリースに紐づく。
- リリースごとの曲順（トラック順）を保持できる。
- 作詞/作曲/編曲/振付を役割ごとに複数人登録できる。
- 衣装を複数登録でき、衣装ごとに担当者と画像を保持できる。

### 3. フォーメーション

- 列数と列ごとの人数を設定できる。
- 参加メンバーは既存 `orbit_members` から選択し、かつ当該リリース参加者から選択する。
- 列ごとの人数設定と割当メンバー数は一致必須。

### 4. MV

- MVあり/なしを管理できる。
- MVありの場合はリンク必須。
- 監督、ロケ地、配信日、メモは任意。

## データモデル方針（論理）

- `releases`: リリース本体
- `tracks`: 楽曲本体
- `release_tracks`: リリースと楽曲の中間（曲順を保持）
- `people`: 人物マスタ（表示名）
- `track_credits`: 楽曲クレジット（role + person）
- `release_members`: リリース参加メンバー
- `track_formations` / `track_formation_members`: フォーメーション列定義と割当
- `track_mvs`: MV情報
- `release_bonus_videos`: 特典映像
- `track_costumes`: 衣装情報

注: 物理テーブル名は既存 `orbit_` 命名規則に合わせて実装時に確定する。

## 画面スコープ

- Admin: リリース/楽曲の一覧・作成・編集・削除（関連子要素含む）
- Public: 一覧/詳細（リリース情報、楽曲詳細、フォーメーション表示）

## 非対象（Non-goals）

- 既存楽曲データの移行
- ライブ/セットリスト実装
- 匿名公開向けRLS方針変更

## 受け入れ条件（実装完了判定）

- 新モデルで Admin CRUD が成立する。
- 新モデルで公開一覧/詳細が成立する。
- 必須項目と制約（タイプ別ナンバリング、MVリンク、フォーメーション整合）が検証される。
- `pnpm --filter oshikatsu-web typecheck` と `pnpm --filter oshikatsu-web lint` が通る。
