# ADR 0013: Sakalog主要閲覧ジャーニーのDaily Story方針

## Status

Accepted

## Context（背景）

Sakalogは、今日の出来事を日々の訪問理由にし、蓄積した思い出と今後の予定をつなぐ
推し活アーカイブを目指している。一方、Design Auditでは、月間カレンダーがトップページで
最も強く見えるため日次価値が伝わりにくいことと、日付から発見したライブの文脈が
該当公演や参加記録まで継続しないことがP1の課題として確認された。

Issue #341では、主要閲覧ジャーニーの基礎を次の3方向から比較した。

### Daily Story

「今日」を第一の閲覧文脈とし、現在の情報を過去の思い出と今後の予定へつなげて読む。
Product Principleの「毎日開く理由」「思い出が育つ」「過去とこれからをつなぐ」を
トップページの体験へ最も直接的に反映できる。一方、月間探索の視認性は相対的に下がる。

### Date Thread

選択日を一貫したthreadとして、イベント、該当公演、参加記録まで強く継承する。
再探索の防止には最も強いが、日付操作やカレンダーがコンテンツより中心に見えやすい。

### Momentum Loop

「次に楽しむ、参加する、残す」という循環を中心に置く。参加状態と参加記録への接続は
明快になるが、閲覧中心のViewerへ行動を促しすぎ、静かなアーカイブ性を弱める可能性がある。

## Decision（決定）

主要閲覧ジャーニーのBase Directionとして **Daily Story** を採用する。

- 「今日」を第一の閲覧文脈とする。ただし、日付そのもの、巨大なHero、詩的なコピーを
  主役にはせず、今日に関係する情報の構成によって日次価値を伝える。
- 月間カレンダーは、主要コンテンツではなく日付から過去や予定を探すための二次探索導線とする。
- Date Thread全体は採用しない。日付から詳細へ進む場合に、選択日と該当performanceの文脈を
  継承し、再探索を避ける原則だけを採用する。
- Momentum Loop全体は採用しない。該当公演、参加状態、参加記録を近接させ、発見から記録へ
  自然に進める原則だけを採用する。
- Viewerへ登録や記録を過度に促さない。白黒中心の静かな情報面と、過去・現在・未来を
  落ち着いて見続けられるアーカイブ性を維持する。

この組み合わせにより、Daily Storyがプロダクトの第一印象と日次閲覧の軸を担い、
Date Threadは文脈の連続性、Momentum Loopは参加情報との局所的な接続を補完する。

## Scope Boundary（仕様境界）

このADRは主要閲覧ジャーニーの判断原則と採用理由を記録する。次の画面仕様は複製せず、
Issue #341のDecisionを参照する。

- Top Page Hierarchyの具体的な順序
- 表示件数
- Desktop / Mobileの具体配置
- Global Navigationの項目
- Detail Contextの具体挙動
- Implementation Split

カレンダーの意味モデル、テーマやinteraction stateのhardening、read modelの変更も
このADRでは決定しない。

## Consequences（結果・影響）

### 良い点

- Product PurposeとDesign Principle 1〜4を、主要閲覧ジャーニーの優先順位へ一貫して反映できる。
- 予定の有無にかかわらず、現在を過去の思い出と今後の予定へつなぐ日次体験を設計できる。
- 日付起点の発見から該当公演・参加記録までの再探索を減らせる。
- 行動喚起を局所化し、Viewerにも適した静かなアーカイブ性を維持できる。

### トレードオフ

- カレンダーを二次導線にするため、月間探索は初期表示で見つけにくくなる場合がある。
- 選択日と該当performanceの文脈を検証し、直接訪問時と区別する状態管理が必要になる。
- 日次価値は情報の構成に依存するため、予定や過去の記録が少ない状態も継続して成立させる必要がある。
- 参加導線を控えめにするため、記録行動を最優先する体験よりコンバージョンは弱くなり得る。

## Review Trigger（見直し条件）

次の場合はこのADRを見直す。

- SakalogのProduct Purposeが、日次閲覧より月間計画または記録行動を優先する方向へ変わる場合
- Viewerと管理者の主要タスクが変わり、静かな閲覧中心の体験が適合しなくなる場合
- 実装・再監査により、Daily Storyでは予定が少ない日の継続体験を成立させられないと確認された場合
- カレンダーの後続Decisionにより、二次探索導線という位置付けとの矛盾が生じる場合
